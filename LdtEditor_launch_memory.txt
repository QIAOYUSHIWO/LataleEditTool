  --- L0 决策归档（固化结论；若变更受众或标准，改此处并复审 L1–L5）---

  L0-1 首发受众
    · 首版定位：**小圈分发**（熟悉表结构的编辑者、工具链同好、可沟通反馈的小范围用户）。
    · **不**将首版目标定为「零说明的公共应用商店级小白用户」；若改为公开广撒下载，须同步提高 L4（签名、安装包）与 L5（文档）要求。
    · L4 代码签名：首版按**暂缓**处理也可接受；公开渠道再强制执行 L4-2。

  L0-2 首发最低标准（与下方 L1/L2/L5 挂钩）
    · **必须收口**：Typed 表写盘与偏移一致性问题 —— 须**修复**或**保存前硬拦 + 明确报错**，禁止静默坏档（对应 L1-1）。
    · **必须定案**：覆盖保存前的备份或可配置安全网（对应 L1-2）；保存失败不得留下难以恢复的半成品（行为需确认或加固）。
    · **必须可验证**：主力 `.ldt` 类型「打开 → 编辑 → 保存 → 再打开」可重复手工验证（与 L2-2 一致）。
    · **可写入「已知问题」**：批量插入无预览/仅初版行为、全局热键与第三方软件冲突、极端大表未做系统压测等。
    · **建议**：功能菜单「插入」批量数值在 UI 或帮助中标为**实验/初版**（非强制，但与用户预期对齐）。

  L0-3 与开发记忆「待完成/可选」对齐 — 首发必做 vs 首发后（各 ≤5 条）
    · **首发必做**
      1. L1-1：Typed / `DataStartOffset`·`DataEndOffset` / `BuildPatchedTypedFile` 一致性（见开发记忆已知项）。
      2. L1-2：备份策略定案与实现（或用户可选开关）。
      3. L3：Release 构建路径 + 运行时策略（framework-dependent 或 self-contained）写进发布说明。
      4. L5 最小集：配置目录、`SPF` 路径、替换须先预览再「直接」、核心快捷键（Ctrl+F / Shift+P / Shift+A）一页说明。
      5. L5-3：首版「已知问题」列表与 L0-2 一致，随版本更新。
    · **首发后做（不阻塞首版打包）**
      1. 批量插入增强：预览、仅筛选可见行、参数记忆、布尔规则可配置等。
      2. 产品向：主窗查找与功能菜单筛选联动、说明列手选记忆、最近文件钉选、安装包级 `.ldt` 关联。
      3. 数据面：LRU/引用调参、单图最大像素或缩略图策略、其它 SPF 链序特例。
      4. L6-4：更新检查或固定发布渠道公告（可选）。
      5. 其余开发记忆中「可选深化」未列入上列者，默认**首发后不紧急**。
