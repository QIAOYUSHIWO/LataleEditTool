**核查说明（与当前程序对照，2026-05-13）**

- **L2-4 命令**：**第一个参数**必须是子命令 `ldt-roundtrip`，**第二个参数**为 LDT 路径；路径含空格时**整段加英文双引号**。第三个参数可选：`gbk`（或省略）与 `cp949`。若首参不是子命令或路径错误，程序会打印 `Usage:` 并**不会**执行 roundtrip（这不是退出码，而是帮助文本）。
- **实测**（供参考）：`ITEM_1 - 副本.LDT` 在仓库 Release 构建下执行  
`LdtEditor.exe ldt-roundtrip "F:\…\ITEM_1 - 副本.LDT"` → **退出码 0**（默认 `gbk`）；  
同一文件加 `**cp949`** → **退出码 1**（语义不一致），说明该表字符串列与 CP949 再编码不一致，**以 GBK 为默认 roundtrip 更匹配此文件**。与 GUI **选项 → 存盘** 编码对齐可减少「另存乱码」。与 README「命令行」节同源。
- **L2-2 乱码**：另存后中文乱码多因 **存盘编码（GBK/CP949）与原始文件不一致**；请在选项中切换存盘编码后重新保存或从 `.bak` 恢复再试。
- **L2-3 热键**：程序以 `RegisterHotKey` 注册 Ctrl+F、Shift+P、Shift+A；**多开实例**时通过线程级 `IMessageFilter` 处理 **WM_ACTIVATEAPP**，在进程整体失前台时卸载热键、回前台后短间隔重试注册，避免非前台实例仍占用热键导致另一实例注册失败却收不到 `WM_HOTKEY`。**无冲突且注册成功时**，焦点在外部窗口仍应能唤起功能菜单 / 预览 / 选图窗。**若某项注册失败**：该项全局热键不可用；**主窗有焦点**时仍应由 `MainForm_KeyDown` 处理 Ctrl+F / Shift+P / Shift+A（与 README 一致）。若实测「主窗有焦点仍不响应」，属待查，请在 issue 中注明焦点控件、是否另有顶层窗体、是否多开。

## L2-4 自动化最小回归（`ldt-roundtrip`）

**用意**：在不改表内逻辑的前提下，检查「解析 → 按指定编码写回行区 → 再解析」是否**逐格语义一致**，用于尽早发现序列化/偏移类静默错误（**不能**替代 L2-2 真实编辑保存验收）。

对**已纳入 L2-2 的样本文件**（或专用测试表）在命令行执行（**请使用 Release 构建的 `LdtEditor.exe` 完整路径**）：

```text
LdtEditor.exe ldt-roundtrip "D:\路径\文件.ldt"
LdtEditor.exe ldt-roundtrip "D:\路径\文件.ldt" gbk
LdtEditor.exe ldt-roundtrip "D:\路径\文件.ldt" cp949
```

- **第一个参数**固定为子命令 `ldt-roundtrip`；**第二个参数**为 LDT 路径（含空格必须加引号）。  
- **第三个参数**可选：`cp949` 使用 CP949 写字符串列；省略或 `gbk` 使用 GBK（代码里省略时与 `gbk` 相同策略）。  
- **退出码**：`0` 通过；`1` 语义不一致（可换 `gbk`/`cp949` 与 GUI 存盘编码对齐后重试）；`2` 文件不存在。  
- 若控制台出现 `**Usage: LdtEditor ...`**：说明**未进入** `ldt-roundtrip`（参数顺序错误或路径未加引号等）。

| ITEM_1 - 副本.LDT | `LdtEditor.exe ldt-roundtrip "F:\LataleS4\LataleS4(9999)\ROWID\DATA\LDT\ITEM_1 - 副本.LDT"` | 0                            | 2026-05-09 |
| 同上              | 同上 + `cp949`                                                                              | 1（该文件与 CP949 再编码不一致，属预期对比结果） | 2026-05-09 |


说明：若表内字符串在「自动解码」与「指定存盘编码」之间不一致，可优先用与客户端一致的 `gbk` / `cp949` 与 GUI **选项 → 存盘** 对齐后再测。
